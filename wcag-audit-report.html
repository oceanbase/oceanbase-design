📋 WCAG 2.1 AA 逐项评估（37 条映射，其中 1 条 N/A）
准则编号 名称 等级 状态 说明
1.1 文本替代
1.1.1 Non-text Content A 部分 antd 图标组件无 alt 文本；@oceanbase/icons 已支持装饰性 aria-hidden 与语义 aria-label；业务自定义图标按钮仍需可访问名称
1.3 可适配性
1.3.1 Info and Relationships A 部分 antd 表单/表格具备语义化结构；@oceanbase/ui 中 BatchOperationBar、BasicLayout 等复合布局需业务侧补充 landmark / aria-* 时关联性不完整
1.3.4 Orientation AA 通过 组件库无固定方向锁定，支持横竖屏
1.3.5 Identify Input Purpose AA 通过 表单组件支持 autocomplete 属性传递
1.4 可辨识性
1.4.1 Use of Color A 部分 若仅用 colorSuccess 等主色块传达状态仍不足；主题提供的 color*Text + 浅色底组合对比充足。Tag/Alert 建议保留图标或文案前缀，避免单色依赖（1.4.1）
1.4.3 Contrast (Minimum) AA 失败 正文级：第三级文本 #8592ad 3.1:1、第四级 #b6c0d4 1.8:1；语义主色作正文时 Success/Warning 低于 4.5:1；Error #eb4242 约 3.9:1。Slider 悬停轨道已改为 blue5 #0852bb（7.2:1）。次要文本 #5c6b8a 已达标。见 可访问性文档 约定用法。
1.4.4 Resize Text AA 通过 antd 使用 rem/em 单位；OceanBase 主题未固定 px 字号；支持 200% 缩放
1.4.10 Reflow AA 通过 组件基于 Flex/Grid 布局，支持 320px 宽度无水平滚动
1.4.11 Non-text Contrast AA 失败 边框 #cdd5e4 / #e2e8f3 相对白底 <3:1；焦点环已由主题 lineWidthFocus: 2 与 global.tsx 兜底轮廓补强（2.4.7 主要风险已降低）
1.4.12 Text Spacing AA 通过 antd 组件支持行高/字间距覆盖；未使用固定容器限制文字扩展
1.4.13 Content on Hover or Focus AA 通过 Tooltip/Popover 支持 hover 和 focus 触发；Popover 可通过 ESC 关闭
2.1 键盘可访问
2.1.1 Keyboard A 部分 antd 核心组件具备键盘导航；@oceanbase/ui 中部分自定义控件使用非 <button> 的可点击元素时可能无法通过 Tab 聚焦；另受 antd DatePicker tab 顺序问题影响（issue #53638）
2.1.4 Character Key Shortcuts A 通过 组件库未使用单字符快捷键；Hotkeys 需组合键触发
2.2 足够时间
2.2.1 Timing Adjustable A 通过 Message/Notification 自动消失支持配置 duration；Tour 步骤无自动推进
2.2.2 Pause, Stop, Hide A 通过 Message/Notification 支持手动关闭；无自动播放媒体内容
2.3 癫痫 / 身体反应
2.3.1 Three Flashes A 通过 无闪烁动画；Loading 动画速率安全；Lottie 动画由业务控制
2.4 可导航
2.4.3 Focus Order A 部分 antd Modal/Drawer 支持 focus trap；@oceanbase/ui 的 Dialog 已增强打开时焦点进入与 Escape 关闭；SideTip 等浮动面板的完整焦点陷阱仍需业务场景验证
2.4.7 Focus Visible AA 通过 默认主题 lineWidthFocus: 2，global.tsx 对非 antd 类名原生控件补充 :focus-visible；建议在业务复杂布局上仍做一次键盘走查
2.4.11 Focus Not Obscured AA 部分 固定表头/底栏可能遮挡键盘焦点；BasicLayout 侧栏折叠控件已改为 button 并带 aria-expanded / aria-label，复杂嵌套滚动区仍需验证
2.5 输入方式
2.5.1 Pointer Gestures A 通过 无复杂手势依赖；拖拽类操作（Table 列排序）提供按钮替代
2.5.2 Pointer Cancellation A 通过 Button 使用 mousedown+click 模式；无按下即生效的长按操作
2.5.7 Dragging Movements A 通过 无纯拖拽功能；Table 列排序提供按钮操作
2.5.8 Target Size (Minimum) AA 部分 Dialog / FullscreenBox 控制按钮已增至 ≥24×24px；Table / ProTable 行内 icon-only 仍可能紧凑，见 可访问性文档 2.5.8 约定

3.1.2 Language of Parts AA N/A 组件内容语言一致，无多语言混合场景

3.2.2 On Input A 通过 表单输入不自动提交；Filter 支持手动触发搜索

🛠️ 修复建议（按优先级排序）
P1
调整主题色对比度
按上文色表：避免将 colorSuccess / colorWarning 直接用作小号正文；第三级文本 colorTextTertiary 仅作大号或辅助说明；正文级状态文案优先使用 color*Text + color*Bg。边框可配合阴影或加粗而不仅依赖色相。
P1
为 @oceanbase/ui 自定义组件添加 ARIA 标注
已对 Dialog、DocDialog、BasicLayout、SideTip、BatchOperationBar、Highlight、NavMenu 等落地 role / aria-*、aria-live 与语义化按钮；请继续为 FooterToolbar、TagSelect、LightFilter 等其余导出按同类模式扩展，并为需要的路由项补充 aria-current="page"。
P3
焦点可见性（已完成基线，持续走查）
默认主题已将 lineWidthFocus 设为 2，global.tsx 为非 antd 类名原生控件提供 :focus-visible；建议在业务侧对复杂嵌套滚动、iframe、Monaco 等场景继续做键盘走查。
P3
图标组件 aria 策略（包装层已落地）
装饰性图标默认 aria-hidden；语义图标通过 aria-label / ariaLabel 透传；业务仍需为带交互的图标按钮提供可访问名称。
P2
Dialog/SideTip 实现 focus trap
复用 antd 的 FocusTrap 或 rc-util 的 focus 工具，确保弹窗打开时焦点锁定在内部，关闭时返回触发元素。
P2
BackgroundTaskManager 添加 aria-live
在任务进度更新时使用 aria-live="polite" 区域通知屏幕阅读器，如 "3/10 任务已完成"。
P3
增大触控目标至 ≥ 24×24px
Table 操作列、ProTable 行内 icon-only 按钮等确保最小可点区域 ≥24×24px（含透明 hit slop）。
P3
为状态色增加非颜色标识
Alert、Tag 组件在颜色之外增加图标前缀，满足 WCAG 1.4.1 "信息不应仅依赖颜色传达"。
📖 WCAG 2.1 AA 速查卡

对比度要求

正文文本 → ≥ 4.5:1
大号文本 (≥18pt / ≥14pt 粗体) → ≥ 3:1
UI 组件 / 图形 → ≥ 3:1

四大原则

Perceivable 可感知 — 信息不可仅依赖颜色
Operable 可操作 — 全键盘可访问、焦点可见
Understandable 可理解 — 语言标记、错误提示
Robust 健壮 — ARIA 标注、屏幕阅读器兼容
